# نظام وكلاء الذكاء الاصطناعي لبناء المواقع (GitHub Actions Edition)

هذا النظام مصمم ليعمل بشكل مستقل تماماً على GitHub، حيث يقوم الوكلاء بالتعلم والإصلاح الذاتي وبناء المواقع بشكل دوري.

## إعدادات GitHub Secrets
لكي يعمل الوكلاء، يجب عليك إضافة مفاتيح Groq API الخاصة بكل وكيل في إعدادات المستودع (Settings > Secrets and variables > Actions):

| اسم الـ Secret | الوكيل المرتبط |
| :--- | :--- |
| `ALPHA_GROQ` | Agent Alpha |
| `BETA_GROQ` | Agent Beta |
| `GAMMA_GROQ` | Agent Gamma |
| `DELTA_GROQ` | Agent Delta |
| `EPSILON_GROQ` | Agent Epsilon |

## كيف يعمل النظام على GitHub؟
1. **التشغيل الدوري**: تم إعداد `GitHub Actions` ليعمل النظام تلقائياً كل ساعة.
2. **التشغيل اليدوي**: يمكنك تشغيل الوكلاء يدوياً من تبويب `Actions` في GitHub.
3. **حفظ التقدم**: بعد كل تشغيل، يقوم الوكلاء بحفظ "المعرفة المكتسبة" و"الذاكرة" و"المواقع المبنية" وإعادة رفعها (Push) إلى المستودع تلقائياً.

## المميزات التقنية
- **استقلالية المفاتيح**: كل وكيل يستخدم مفتاحه الخاص، مما يمنع تداخل الحصص (Quotas) ويزيد من الأمان.
- **التواصل عبر الملفات**: يقرأ الوكلاء ويكتبون في `communication_hub/messages/log.txt` لتبادل الخبرات.
- **الإصلاح الذاتي**: إذا واجه أحد الوكلاء خطأ في GitHub Action، سيحاول البحث عن حل واستخدامه في التشغيل القادم.

## هيكل المشروع المحدث
- `.github/workflows/agents_swarm.yml`: ملف التكوين الخاص بالتشغيل الآلي.
- `agents/`: مجلدات الوكلاء الخمسة (كل واحد بملفه الخاص).
- `shared_tools/`: محرك الوكلاء، أدوات البحث، والتعلم.
- `knowledge_base/`: قاعدة المعرفة التي تنمو مع الوقت.

## ملاحظة هامة
تأكد من إعطاء صلاحيات الكتابة لـ GitHub Actions في مستودعك (Settings > Actions > General > Workflow permissions > Read and write permissions).
